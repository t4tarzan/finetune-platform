finetune-platform deployed as release "{{ .Release.Name }}" in namespace "{{ .Release.Namespace }}".

It runs as a SINGLE stateful pod (UI :{{ .Values.ports.ui }} + inference :{{ .Values.ports.inference }})
using the HuggingFace/PyTorch CPU backend.

1) Wait for it to come up (first boot pulls the base model — can take minutes):
   kubectl -n {{ .Release.Namespace }} rollout status deploy/{{ include "finetune-platform.fullname" . }}

2) Open the UI:
{{- if .Values.ingress.enabled }}
   http{{ if .Values.ingress.tls }}s{{ end }}://{{ .Values.ingress.host | default "<ingress-address>" }}/
{{- else }}
   kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "finetune-platform.fullname" . }} {{ .Values.ports.ui }}:{{ .Values.ports.ui }}
   open http://localhost:{{ .Values.ports.ui }}
{{- end }}

{{- if .Values.ollama.enabled }}

3) Pull a base model into the Ollama sidecar (for base-model chat):
   kubectl -n {{ .Release.Namespace }} exec -c ollama deploy/{{ include "finetune-platform.fullname" . }} -- ollama pull qwen2.5:0.5b
{{- end }}

NOTE: CPU training is slow — use small models (0.5B / 1.5B) and small datasets on a
CPU node like m5.xlarge. For real throughput, deploy on a GPU node group with a
CUDA image (see chart README).
