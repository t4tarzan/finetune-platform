🚀 Fine-Tuning Appliance — Deploy on Kubernetes

A self-contained SRE fine-tuning app: bundled data, base models, and a pre-trained model — installed with one Helm command, runs fully offline.

Self-contained Air-gapped amd64 / CPU 2 commands

What you get

Baked Everything in the image

Base models (Qwen2.5 0.5B + 1.5B), 43 SRE observability tables, training data, and a pre-trained sre-assistant — no downloads.

Cards Query your data

One-tap preset cards (pods at risk, OOM offenders…) and a "Query data" mode that writes SQL from plain English.

Tune Train & retrain

Fine-tune on bundled data, then Continue from fine-tuned on more data — the answers improve.

Yours Bring your own data

Upload a CSV/JSONL — it converts and lands in the dropdown, ready to retrain on.

How it runs

One stateful pod. The base models, data, and adapter are baked into the image; an optional Ollama sidecar (also baked) provides base-model chat. Nothing is downloaded at runtime — the only network use is pulling the two public images once.

1

Install — one command

Point kubectl at your cluster (with a gp3 StorageClass), get the chart, and install:

git clone https://github.com/t4tarzan/finetune-platform.git
cd finetune-platform

helm install finetune-platform charts/finetune-platform \
  --namespace finetune --create-namespace \
  --set persistence.storageClass=gp3

The image (ghcr.io/t4tarzan/finetune-platform:latest, amd64) and the baked Ollama sidecar are the chart defaults — nothing else to set.

Brand-new to AWS? The AWS-CONSOLE-GUIDE.md has a full click-by-click version using AWS CloudShell (no software on your computer) — log in, open CloudShell, paste a few commands, open a link. Use a m5.2xlarge node.
2

Open it

kubectl -n finetune rollout status deploy/finetune-platform   # first boot pulls the images
kubectl -n finetune port-forward svc/finetune-platform 7100:7100
# open http://localhost:7100

Or expose a URL with --set service.type=LoadBalancer.

Try it

  • Chat tab → tap a card (e.g. Top OOM offenders) → a table from your data.
  • Tick 🗄️ Query data, ask top 5 namespaces by alert count → it writes the SQL and answers.
  • Pick sre-assistant in the model dropdown → ask an SRE question.
  • Train tab → pick sre_qa_v1 → train → Export & Serve → then sre_qa_v2 with Continue from fine-tuned → retrain → the answer improves.
  • ⬆ Upload CSV / JSONL → bring your own data → retrain on it.

Air-gapped clusters

The app needs no internet at runtime — only to pull two public images once. Mirror them into your private registry and set --set image.repository=<your-registry>/finetune-platform (and ollama.image); then it runs completely offline.
