🦾 Fine-Tuning Platform v1.0

Local recursive fine-tuning platform for Apple Silicon — MLX-native, multi-model consensus, live training UI, and Ollama serving. Built for the M5 Ultra with 128GB unified memory.

Consensus Multi-Model Verification

4 diverse LLMs verify each training row. Only data with 3+ agreements passes. Eliminates hallucination propagation.

MLX Apple-Native Training

MLX LoRA fine-tuning on Apple Silicon. Uses unified memory directly — no CUDA, no translation layer.

SSE Live Progress

Server-Sent Events stream loss, learning rate, tokens/sec, and ETA in real-time during training.

Ollama One-Click Serving

Fine-tuned models auto-register with Ollama and appear in the chat dropdown immediately.

Leaderboard Benchmark Tracking

Automated eval on held-out test sets. Tracks accuracy, grounding, and consistency with baseline deltas.

Recursive Improvement Loop

Generate → verify → train → eval → find gaps → repeat. Stops at convergence or max iterations.

Quick Start

git clone https://github.com/YOUR_ORG/finetune-platform.git
cd finetune-platform
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
brew install ollama && brew services start ollama
python ui/app.py

Open http://localhost:7100 — two tabs: Chat (inference) and Train (fine-tuning).

Architecture

Browser (localhost:7100)
  ├── Chat Tab  ─── /api/chat ─── Ollama (inference)
  └── Train Tab ─── /api/train/* ─── SSE Progress ─── Subprocess Worker (MLX LoRA)
                      ├── Consensus Verification (4 models)
                      ├── Training Manager (state machine)
                      └── Eval Harness + Leaderboard

API Endpoints

GET  /api/models              — List Ollama models
GET  /api/chat                — Chat with a model
POST /api/train/start         — Start training
GET  /api/train/status        — Training state + metrics
GET  /api/train/progress      — SSE progress stream
POST /api/train/stop          — Stop training
GET  /api/train/history       — Past runs
POST /api/export              — Export to Ollama
GET  /api/leaderboard         — Benchmark data
GET  /api/docs/v1             — Full JSON documentation

Configuration

All settings in pipeline/config.yaml:

base_model: "mlx-community/Qwen2.5-7B-Instruct-4bit"
consensus_models:
  - "deepseek/deepseek-v4-pro"
  - "Qwen/Qwen3.6-Max-Preview"
  - "moonshotai/Kimi-K2.7-Code"
  - "claude-sonnet-4-6"
consensus: { min_agree: 3, confidence_threshold: 0.7 }
training: { lora_rank: 16, learning_rate: 1e-4, batch_size: 4, epochs: 3 }
ports: { chat_ui: 7100 }

Hardware

  • Apple M5 Max + 128GB unified memory
  • MLX Metal: available — no CUDA required
  • Fits 70B 4-bit model (~45GB) + TurboVec index (~4GB) + training cache simultaneously
  • Works on any M-series Mac (M1/M2/M3/M4/M5)

Data Flow

  1. Generate — BigSet or local JSONL produces raw dataset
  2. Verify — 4 models check each row; ≥3 agreements required
  3. Split — Verified data → train.jsonl / valid.jsonl
  4. Fine-tune — Isolated subprocess runs MLX LoRA, streams progress
  5. Export — Adapter merged → GGUF → Ollama registration
  6. Chat — Model appears in dropdown, ready for inference
  7. Repeat — Eval failures inform next dataset iteration
